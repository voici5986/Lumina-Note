# Lumina Note 使用流程说明（中文）

这是一份面向新用户的流程化说明，目标是用最短路径跑通核心能力，而不是覆盖所有功能细节。

## 适合谁

- 第一次使用 Lumina Note 的用户
- 想快速理解“编辑器 + AI + 知识图谱 + PDF”如何串起来的用户

## 5 分钟上手

### 前置准备

- 已安装应用（从 Releases 下载）
- 已有一个本地文件夹可作为笔记库
- 已准备模型 API Key（OpenAI / Claude / Gemini 等）

### 步骤

1. 启动应用并选择本地文件夹作为笔记库。
2. 在右侧 AI 面板选择提供商并填写 API Key。
3. 新建一条笔记，输入一些标题和正文。
4. 在正文里创建一个 `[[双链]]`，再打开被链接页面补充内容。
5. 打开知识图谱，确认两条笔记已经建立连接。

### 你应该看到的结果

- 能正常创建、编辑、保存 Markdown 笔记
- AI 面板可正常对话
- 知识图谱中出现你创建的连接关系

## 工作流 A：日常笔记整理

### 目标

把零散记录整理成可检索、可关联的知识网络。

### 推荐步骤

1. 在当天笔记中快速记录信息（不用一开始追求结构）。
2. 用 `[[双链]]` 把内容关联到已有主题页。
3. 在 AI 面板中让 Agent 帮你归纳成小节和待办。
4. 打开知识图谱检查是否出现“孤立笔记”，补齐关联。

### 常见问题

- 图谱里节点太乱：先按主题拆分笔记，再补链接。
- AI 回答偏泛：在提问里明确“只基于当前笔记库”。

## 工作流 B：PDF -> 可检索笔记

### 目标

把 PDF 阅读标注转成可长期复用的 Markdown 知识。

### 推荐步骤

1. 在 PDF 阅读器中高亮和批注关键段落。
2. 将批注导出或保存到对应 Markdown 页面。
3. 在该页面补充你的结论和标签。
4. 用 AI 让其只基于这页内容生成摘要或问答卡片。

### 常见问题

- 标注很多但难检索：给每次阅读建立独立主题页并统一命名。
- 摘要不准确：先限定“只总结高亮段落”再扩展。

## 工作流 C：用 Agent/Codex 做重构

### 目标

让 AI 帮你做结构化改写，而不是一次性重写全文。

### 推荐步骤

1. 先圈定操作范围（当前文档 / 某个目录）。
2. 给出明确任务（例如“重排目录并保留原事实”）。
3. 让 AI 先给变更计划，再执行改写。
4. 人工复核关键段落后再保存。

### 建议

- 优先做“小步修改 + 多轮确认”，稳定性更高。
- 对重要内容要求 AI 给出“改写依据”，便于审查。

## 数据与隐私边界

- 笔记默认本地存储（local-first）。
- 使用云模型时，发送到模型服务的数据受你当前提问和配置影响。
- 在生产场景中，建议为不同数据敏感级别使用不同模型配置。

## FAQ（简版）

### 1. AI 没有回复怎么办？

- 检查 API Key 是否有效。
- 检查模型名称与提供商配置是否匹配。
- 检查网络连通性或代理设置。

### 2. 检索结果不理想怎么办？

- 增加明确标题和小节，减少超长段落。
- 用统一标签和双链提升语义关联。

### 3. 刚开始应该优先学什么？

- 先掌握：`双链` + `AI 面板` + `知识图谱` 三件事。
- 其余高级功能按需开启即可。
