# Lumina Note ユーザーガイド（日本語）

このガイドは、新規ユーザーが最短で実用的なワークフローに入るためのものです。機能一覧の完全版ではありません。

## 対象ユーザー

- Lumina Note を初めて使う人
- エディタ / AI / ナレッジグラフ / PDF の連携を素早く掴みたい人

## 5 分セットアップ

### 事前準備

- Releases から Lumina Note をインストール済み
- ローカルのノート保管フォルダ（Vault）
- モデルプロバイダの API Key（OpenAI / Claude / Gemini など）

### 手順

1. アプリを起動し、ローカルフォルダを Vault として選択する。
2. 右側の AI パネルでプロバイダと API Key を設定する。
3. 最初のノートを作成して内容を入れる。
4. `[[WikiLink]]` で別ページにリンクし、リンク先にも内容を入れる。
5. ナレッジグラフを開き、ノート同士が接続されていることを確認する。

### 期待される状態

- Markdown ノートを作成 / 編集 / 保存できる
- AI チャットが使える
- グラフ上でノート間の関係が見える

## ワークフロー A: デイリーノートから構造化知識へ

### 目的

散在するメモを、検索しやすく関連付け可能な知識に変える。

### 手順

1. まずはデイリーノートに素早く記録する。
2. 重要な記述を `[[WikiLinks]]` でテーマページへ接続する。
3. Agent にセクション整理やアクション項目の抽出を依頼する。
4. グラフで孤立ノートを見つけ、リンクを追加する。

## ワークフロー B: PDF から再利用可能なノートへ

### 目的

読書ハイライトを長期利用できる Markdown 資産へ変換する。

### 手順

1. PDF Reader でハイライトと注釈を付ける。
2. 注釈を Markdown ページへ保存 / エクスポートする。
3. 自分の結論とタグを追記する。
4. そのページのみを根拠に AI 要約を作らせる。

## ワークフロー C: Agent/Codex 支援のリファクタ

### 目的

一発書き換えではなく、制御された構造変更を行う。

### 手順

1. 先に対象範囲を決める（現在ファイル / 特定フォルダ）。
2. 具体的な指示を出す（例: 「事実は維持して見出しを再構成」）。
3. まず計画を出させ、その後に編集を適用する。
4. 最終保存前に重要箇所を手動で確認する。

## プライバシーとデータ境界

- ノートはデフォルトでローカル優先（local-first）。
- クラウドモデルへの送信内容は、プロンプトとプロバイダ設定に依存する。
- 機密データは、用途ごとにモデル / 設定を分離して扱うことを推奨。

## FAQ（短縮版）

### AI が応答しない

- API Key の有効性を確認する
- モデル / プロバイダ設定の整合性を確認する
- ネットワーク / プロキシ設定を確認する

### 検索品質が弱い

- 見出しを明確にし、セクションを短くする
- タグと WikiLinks を一貫して付ける

### 最初に覚えるべきこと

- `WikiLinks + AI パネル + ナレッジグラフ` の 3 点から始める
